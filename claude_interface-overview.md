## 🎨 인터페이스 종류 한눈에 보기

Claude Code는 **하나의 두뇌(CLI 엔진)** 에 **여러 개의 얼굴(인터페이스)** 을 가지고 있어요.

| 인터페이스 | 실행 방법 | 특징 | 추천 대상 |
|------------|-----------|------|-----------|
| 🖤 **터미널 CLI** | `claude` | 가장 강력, 모든 기능 사용 가능 | 핵심·기본 |
| 🟦 **VS Code / Cursor 확장** | Extension 설치 | 그래픽 채팅 패널, 시각적 diff | 일반 개발자 |
| 🟧 **JetBrains 플러그인** | Plugin 설치 | IDE Diff 뷰어 통합 | Java/Python 개발자 |
| 🌐 **웹 인터페이스** | claude.ai/code | 브라우저, 설치 불필요 | 빠른 시연·모바일 |

> 💡 **모두 같은 엔진을 공유**해요. 같은 대화도 인터페이스를 바꿔가며 이어갈 수 있어요!

## 1️⃣ 터미널 CLI 인터페이스 (가장 기본)

`claude` 명령어를 실행하면 보이는 화면이에요. 터미널 안에서 **React 기반 UI**가 그려져요.

### 🗺️ 전체 화면 구조

```
┌─────────────────────────────────────────────────────────┐
│  🤖 Claude Sonnet · /Users/nara/my-project · main       │  ← ① 헤더
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Human: 이 프로젝트의 구조를 설명해줘                    │
│                                                         │  ← ② 메시지
│  ⏺ Read(README.md)                                     │     히스토리
│    ⎿ 읽기 완료 (152줄)                                  │     영역
│                                                         │
│  Assistant: 이 프로젝트는 Next.js로 만든...              │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  > 다음 질문을 입력하세요...                             │  ← ③ 입력창
│                                                         │
├─────────────────────────────────────────────────────────┤
│  ◐ Plan Mode  ·  Opus 4.7  ·  ⚡ 45% context           │  ← ④ 상태 표시줄
└─────────────────────────────────────────────────────────┘
```

### 영역별 설명

#### ① 헤더 영역 (상단)

```
🤖 Claude Sonnet · /Users/nara/my-project · main
```

세 가지 핵심 정보를 한 줄에 보여줘요:

| 표시 | 의미 |
|------|------|
| 🤖 모델 이름 | 현재 사용 중인 Claude 모델 (Opus / Sonnet / Haiku) |
| 📁 작업 경로 | Claude가 인식하는 프로젝트 루트 |
| 🌿 Git 브랜치 | 현재 체크아웃된 브랜치 이름 |

> 💡 **헤더만 봐도 "지금 어디서, 누가, 무슨 브랜치로 일하는지"** 즉시 알 수 있어요.

#### ② 메시지 히스토리 영역 (중앙·가장 넓음)

여기에 대화 내용이 차곡차곡 쌓여요. 특별한 표시들이 있어요:

```
Human:                            ← 내가 쓴 말
Assistant:                        ← Claude의 답변
⏺ Read(파일명)                    ← Claude가 도구를 사용한 기록
⎿ 결과 요약                       ← 도구 실행 결과
```

**아이콘 의미:**

| 아이콘 | 뜻 | 예시 |
|--------|-----|------|
| `⏺` | Claude가 도구 실행 중/완료 | `⏺ Read(app.py)` |
| `⎿` | 도구 실행 결과 (들여쓰기) | `⎿ 152줄 읽음` |
| `✻` | 확장 사고 (Extended Thinking) | `✻ Thinking...` |
| `⠹` `⠼` | 진행 중 (스피너 애니메이션) | 응답 생성 중 |

#### ③ 입력창 (하단)

```
> 여기에 질문이나 명령어를 입력...
```

- **Enter**: 메시지 전송
- **Ctrl+J** 또는 **`\` + Enter**: 줄바꿈 (모든 터미널에서 동작)
- **Shift+Enter**: 줄바꿈 (`/terminal-setup` 실행 후)

#### ④ 상태 표시줄 (맨 아래)

화면 맨 아래에 실시간 정보가 표시돼요:

```
◐ Plan Mode  ·  Opus 4.7  ·  ⚡ 45% context
```

| 표시 | 의미 | 클릭하면? |
|------|------|-----------|
| **권한 모드** | Plan / Default / Auto | 모드 변경 |
| **현재 모델** | 사용 중인 Claude 모델 | `/model` 단축 |
| **컨텍스트 사용량** | 토큰 사용 비율 | `/compact` 추천 |

> ⚠️ 컨텍스트가 **80%를 넘으면** `/compact`로 압축하거나 `/clear`로 초기화하는 게 좋아요!

## 2️⃣ 화면에 보이는 핵심 요소들

### 🎭 권한 모드 인디케이터

화면 어딘가에 항상 표시되는 **현재 모드**예요. 모드에 따라 Claude의 동작이 완전히 달라져요.

| 아이콘 | 모드 | Claude의 행동 | 추천 상황 |
|--------|------|---------------|-----------|
| `◐` | **Plan Mode** | 계획만 보여주고 승인 대기 | 🎓 강의·시연 |
| `○` | **Default** | 매 작업마다 권한 묻기 | 일반 작업 |
| `●` | **Accept Edits** | 파일 편집은 자동 승인 | 빠른 반복 작업 |
| `⚡` | **Auto Mode** | 모든 작업 자동 (위험!) | 신뢰하는 자동화 |

**전환 방법:**
- 화면 하단 모드 인디케이터 **클릭**
- 키보드: **Shift + Tab**
- 명령어: `/permissions`

### 📊 컨텍스트 사용률 표시

```
⚡ 45% context        ← 안전
⚡ 75% context        ← 슬슬 정리 필요
⚡ 92% context  ⚠️    ← 곧 자동 압축 발생
```

> 💡 **컨텍스트** = Claude가 한 번에 기억할 수 있는 대화·코드의 총량

### 🔧 도구 사용 표시 (Tool Use)

Claude가 작업 중일 때 보이는 표시들:

```
⏺ Read(src/app.py)              ← 파일 읽는 중
⏺ Write(src/utils.py)           ← 파일 쓰는 중
⏺ Bash(npm test)                ← 명령어 실행 중
⏺ Edit(README.md)               ← 파일 편집 중
⏺ Glob(*.tsx)                   ← 패턴으로 파일 찾는 중
⏺ Grep("function login")        ← 코드 검색 중
⠹ WebSearch(...)                ← 웹 검색 중 (스피너)
```

> 🎓 **강의 팁**: "Claude가 지금 뭘 하고 있나" 학습자에게 보여주기 좋은 영역이에요!

## 3️⃣ 상호작용 영역 (입력창)

입력창은 **그냥 텍스트만 입력하는 곳이 아니에요**. 특수 기능이 가득해요!

### ⌨️ 4가지 입력 모드

```
> 안녕                               ← 일반 질문
> /help                              ← 슬래시 명령어
> @src/app.py 이 파일 분석해줘       ← 파일 첨부
> !npm test                          ← 셸 명령 직접 실행
```

| 첫 글자 | 모드 | 자동완성 |
|---------|------|----------|
| 글자 | **일반 프롬프트** | - |
| `/` | **슬래시 명령** | 명령어 메뉴 표시 |
| `@` | **파일/터미널 참조** | 경로 자동완성 |
| `!` | **셸 명령 직접 실행** | - |

### 🖼️ 이미지 붙여넣기

```
Ctrl+V (또는 Cmd+V)
```

스크린샷, 디자인 시안, 에러 이미지 등을 **클립보드에서 바로 붙여넣을 수 있어요!**

```
> [Image #1: error.png] 이 에러 메시지 어떻게 해결해?
```

> 🎓 **강의 활용**: UI 버그 보여주기, 디자인 시안 → 코드 변환 시연에 좋아요.

### 📋 긴 텍스트 붙여넣기 처리

10,000자 이상 붙여넣으면 자동으로 축약돼서 표시돼요:

```
> [Pasted text: 5,234 lines]   ← 입력창에는 이렇게 보임
                                  실제로는 전체 내용이 Claude에게 전달됨
```

## 4️⃣ 권한 다이얼로그 이해하기

Default 모드에서 Claude가 작업하려고 할 때 나타나는 화면이에요. **이게 Claude Code의 안전장치예요!**

### 📋 권한 요청 화면 예시

```
┌──────────────────────────────────────────────────┐
│  ⚠️  Claude wants to run:                         │
│                                                  │
│      Bash(rm -rf node_modules)                   │
│                                                  │
│  This will delete the node_modules folder.       │
│                                                  │
│  [ Y ] Yes, run once                             │
│  [ A ] Yes, and don't ask again for `Bash(rm*)`  │
│  [ N ] No, give feedback                         │
└──────────────────────────────────────────────────┘
```

### 선택지 의미

| 키 | 동작 | 언제 쓰나? |
|-----|------|-----------|
| **Y** | 이번 한 번만 허용 | 가끔 쓰는 명령 |
| **A** | 앞으로 같은 종류는 자동 허용 | 자주 쓰는 안전한 명령 (예: `git status`) |
| **N** | 거부 + Claude에게 피드백 | 위험하거나 의도와 다를 때 |

### ⚠️ 위험한 명령어는 강조 표시

빨간색이나 ⚠️ 표시가 붙는 명령어들:

```
⚠️ rm -rf <폴더>           ← 영구 삭제
⚠️ git push -f             ← 강제 푸시
⚠️ DROP TABLE              ← DB 테이블 삭제
⚠️ sudo <명령>             ← 시스템 권한
```

> 🎓 **학습자에게 강조**: 빨간색 명령은 **무조건 한 번 더 읽고** 결정하기!

## 5️⃣ VS Code 확장 인터페이스

VS Code/Cursor에서는 **그래픽 채팅 패널** 형태로 동작해요.

### 🗺️ VS Code 화면 구조

```
┌────────┬─────────────────────────┬────────────────┐
│        │                         │                │
│ 파일   │     ▶ 코드 에디터        │  💬 Claude     │
│ 탐색기 │                         │  채팅 패널      │
│        │  function hello() {     │                │
│        │    console.log("hi")    │  Human: 이...  │
│        │  }                      │                │
│        │                         │  ⏺ Read(...)  │
│        │                         │                │
├────────┴─────────────────────────┤  Assistant:    │
│  🖥️ 통합 터미널                  │  이 함수는...  │
│  $ npm run dev                  │                │
│                                 │  > [입력창]     │
└─────────────────────────────────┴────────────────┘
```

### ✨ 터미널과 다른 점

| 기능 | 터미널 CLI | VS Code 확장 |
|------|-----------|--------------|
| 채팅 인터페이스 | 텍스트 기반 | **그래픽 패널** |
| Diff 보기 | 터미널 텍스트 | **VS Code 네이티브 diff** |
| 파일 첨부 | `@경로` 입력 | **드래그 앤 드롭** |
| 멀티 세션 | 여러 터미널 | **새 탭/창으로 열기** |
| 히스토리 검색 | 위/아래 키 | **검색창 UI** |
| 계획 검토 | 텍스트 출력 | **마크다운 문서로 열기 + 인라인 코멘트** |

### 🎯 핵심 단축키

| 단축키 (Mac / Win·Linux) | 기능 |
|--------------------------|------|
| `Cmd+Esc` / `Ctrl+Esc` | 에디터 ↔ Claude 패널 토글 |
| `Cmd+Shift+P` / `Ctrl+Shift+P` | 명령 팔레트 |
| `/` (입력창에서) | 명령 메뉴 열기 |

> 💡 **선택 영역 자동 인식**: 코드를 드래그로 선택한 뒤 `Cmd+Esc`를 누르면 자동으로 컨텍스트에 포함돼요!

## 6️⃣ JetBrains 플러그인 인터페이스

IntelliJ, PyCharm, WebStorm에서는 **IDE 우측 툴 윈도우**로 통합돼요.

### 🗺️ JetBrains 화면 구조

```
┌──────────────┬─────────────────────────┬───────────┐
│              │                         │           │
│  📁 Project  │   ▶ 코드 에디터          │  🤖       │
│              │                         │  Claude   │
│              │  public class App {     │  Code     │
│              │    public static...     │  [Beta]   │
│              │  }                      │           │
│              │                         │  Human:.. │
│              │                         │           │
├──────────────┴─────────────────────────┤  Assistant│
│  🖥️ Terminal                          │  : ...    │
│  PS> claude                           │           │
│                                       │  > 입력    │
└───────────────────────────────────────┴───────────┘
```

### 🎯 JetBrains만의 특징

| 기능 | 설명 |
|------|------|
| **네이티브 Diff 뷰어** | JetBrains 특유의 강력한 3-way merge 도구로 변경사항 검토 |
| **자동 선택 컨텍스트** | 에디터에서 선택한 코드가 자동으로 Claude에게 전달 |
| **파일 참조 단축키** | `Cmd+Option+K` / `Alt+Ctrl+K` 로 `@File#L1-99` 자동 삽입 |
| **진단 정보 자동 공유** | ESLint, TypeScript, Inspections 결과 자동 인식 |

### ⌨️ 핵심 단축키

```
Cmd+Esc / Ctrl+Esc          : Claude Code 빠른 실행
Cmd+Option+K / Alt+Ctrl+K   : 파일 참조 삽입
Alt+F12                     : 통합 터미널 열기
```

## 7️⃣ 테마 & 외형 커스터마이징

### 🎨 테마 변경

```
/theme
```

내장 테마 중 선택:

| 테마 | 추천 상황 |
|------|-----------|
| **Dark** | 일반 작업 (눈 편함) |
| **Light** | 강의·발표 (스크린샷·녹화) |
| **Auto** | OS 설정 따라가기 |
| **High Contrast** | 시각 보조 필요 시 |
| **Custom** | 직접 만들기 (v2.1.118+) |

### 📊 커스텀 상태 표시줄

화면 하단에 보고 싶은 정보를 추가할 수 있어요:

```bash
# 예시: 현재 모델 + Git 브랜치 + 시간 표시
```

`/config` 또는 settings.json에서 statusLine을 설정해요. 예시:

```json
{
  "statusLine": {
    "type": "command",
    "command": "echo \"$MODEL · $CWD · $(git branch --show-current)\""
  }
}
```

### 🖥️ 풀스크린 모드 (깜빡임 방지)

긴 대화에서 화면이 깜빡이면:

```
/tui fullscreen
```

> 💡 항상 풀스크린으로 시작하려면: `export CLAUDE_CODE_NO_FLICKER=1`

### 🔔 알림 설정

오래 걸리는 작업 끝났을 때 알림 받기:

| 터미널 | 데스크탑 알림 지원 |
|--------|-------------------|
| Ghostty | ✅ 자동 |
| Kitty | ✅ 자동 |
| iTerm2 | ✅ 설정 필요 |

## 🎯 실습: 인터페이스 직접 둘러보기

학습자와 함께 실행해볼 수 있는 시나리오예요. **순서대로 따라가면서 각 영역을 짚어주세요!**

### Step 1. 빈 폴더에서 시작

```bash
mkdir claude-tour && cd claude-tour
claude
```

→ **헤더 확인**: 현재 폴더 경로가 표시되는지 확인

### Step 2. 상태 확인

```
/status
```

→ 인증 방식, 모델, 권한 모드, 컨텍스트 사용량 모두 한 번에 표시

### Step 3. 권한 모드 살펴보기

```
Shift + Tab
```

→ 모드가 순환되는 것을 확인 (Default → Plan → Accept Edits)

### Step 4. 입력창 4가지 모드 시연

```
> 안녕하세요                  (일반)
> /help                       (슬래시 명령)
> @README.md                  (파일 참조 — 자동완성 보기)
> !ls -la                     (셸 명령)
```

### Step 5. 도구 사용 표시 보기

```
> 이 폴더에 README.md 파일을 만들어줘
```

→ `⏺ Write(README.md)` 가 실시간으로 나타남

### Step 6. 권한 다이얼로그 경험

Default 모드에서:

```
> 이 폴더의 모든 파일 목록을 보여줘
```

→ Bash 권한 요청 다이얼로그 → Y/N/A 선택 시연

### Step 7. 테마 바꿔보기

```
/theme
```

→ 화살표로 테마 선택 후 Enter

### Step 8. 컨텍스트 압축 시연

```
> 이 프로젝트에 대해 길게 설명해줘
> 이제 다른 주제로 넘어가자
/compact
```

→ 상태 표시줄의 컨텍스트 % 변화 관찰

## 💡 인터페이스 사용 팁

### ✅ 권장 습관

- 🔍 **`/status` 자주 확인**: 헷갈릴 때마다 현재 상태 점검
- 🧹 **새 주제는 `/clear`**: 컨텍스트 깨끗하게 시작
- 📋 **긴 대화는 `/compact`**: 80% 넘기 전에 압축
- 🎯 **시연은 Plan 모드**: 학습자가 사고 과정을 볼 수 있음

### ❌ 자주 하는 실수

- 🚫 **Ctrl+C로 중단**: 세션이 종료돼요! 응답만 멈추려면 **Esc** 사용
- 🚫 **컨텍스트 100% 방치**: 자동 압축이 일어나며 중요한 정보 손실 가능
- 🚫 **Auto 모드 무분별 사용**: 강의·시연에서는 항상 Plan 모드 권장
- 🚫 **Always(A) 남발**: 위험한 명령까지 자동 허용될 수 있음

### 🎓 강의 시연용 추천 설정

```bash
# 1. 라이트 테마 (스크린샷 잘 보임)
/theme → Light

# 2. Plan 모드 시작
claude --permission-mode plan

# 3. 한국어 응답
claude --append-system-prompt "모든 응답과 코드 주석을 한국어로 작성"

# 4. 풀스크린 (화면 깔끔)
/tui fullscreen
```

## 🆚 인터페이스 비교 정리

언제 어떤 인터페이스를 써야 할까요?

| 상황 | 추천 인터페이스 | 이유 |
|------|-----------------|------|
| 🚀 **빠른 자동화** | 터미널 CLI | 가장 빠르고 모든 기능 제공 |
| 👨‍💻 **일반 개발 작업** | VS Code 확장 | 시각적 diff, 편집기 통합 |
| ☕ **Java/Spring 작업** | JetBrains 플러그인 | IDE 디버거·인덱스 활용 |
| 📱 **외부에서 빠르게** | claude.ai/code (웹) | 설치 불필요 |
| 🎓 **강의·시연** | 터미널 + Plan 모드 | 사고 과정 단계별 노출 |
| 🤖 **CI/CD 자동화** | 터미널 (`-p` 모드) | 비대화형 스크립트 |

> 💡 **세 가지 다 설치하고 상황에 따라 골라 쓰세요.** 같은 대화도 인터페이스를 옮겨가며 이어갈 수 있어요!
