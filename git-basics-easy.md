# Git, 택배처럼 이해하기 📦

> 처음 Git을 배우는 중학생을 위한 가이드입니다.  
> Git이 어렵게 느껴진다면, **"택배 보내기"** 라고 생각해보세요!

---

## 🎯 큰 그림 한 장으로 보기

Git을 한마디로 하면? **"내 코드를 인터넷 창고(GitHub)에 안전하게 보내는 시스템"** 이에요.

```
   [내 책상]      →    [택배 박스]    →    [우리집 보관함]    →    [GitHub 창고]
  내 컴퓨터 폴더        담을 준비           내 컴퓨터에 저장          전 세계 공개

   파일 만들기    →     git add       →       git commit       →      git push
```

이걸 4단계로 천천히 따라가 볼게요! 🚀

---

## 🍱 도시락에 비유하면?

Git을 도시락 만들어 친구에게 보내는 일에 비유하면 이렇게 돼요:

| Git 단계 | 도시락 비유 | 일어나는 일 |
|----------|-------------|-------------|
| 파일 만들기 | 🥪 음식 만들기 | 김밥, 계란말이 만들기 |
| `git add` | 🍱 도시락통에 담기 | "이건 보낼 거야!" 하고 통에 담기 |
| `git commit` | 🎀 포장하고 메모 붙이기 | "친구 생일 도시락" 라벨 붙이기 |
| `git push` | 📮 택배로 부치기 | 친구한테 실제로 보내기 |

> 💡 **핵심**: 만들었다고 바로 보내지는 게 아니에요!  
> **고르고 → 포장하고 → 보내는** 3단계를 거쳐야 해요.

---

## 1️⃣ 작업 폴더로 이동하기

```bash
cd /c/Users/nara/claude-code-guide
```

### 🤔 이게 뭐예요?

`cd` = **C**hange **D**irectory (폴더 바꾸기)

### 🎒 비유하자면?

**"작업할 책상으로 이동하는 것"** 이에요.

```
📍 지금 위치: 거실 (홈 폴더)
   ↓ cd 명령어로 이동
📍 이동 완료: 내 방 책상 (claude-code-guide 폴더)
```

이제부터 이 폴더 안에서 일어나는 모든 일이 Git에 기록돼요. 마치 일기장을 펼치고 오늘 날짜 페이지로 넘기는 것과 같아요! 📔

---

## 2️⃣ 새 파일 만들기

```bash
echo "# 테스트 파일" > TEST.MD
```

### 🤔 이게 뭐예요?

- `echo "..."` = **"이 글자를 화면에 보여줘"**
- `>` = **"근데 화면 말고 파일에 저장해줘"**
- `TEST.MD` = **만들 파일 이름**

### ✏️ 비유하자면?

**"빈 종이에 제목 쓰기"** 와 같아요.

```
📄 빈 종이 ─[ echo ]─→ 📝 "# 테스트 파일" 이라고 쓴 종이
```

### 💡 `.md`가 뭐예요?

**Markdown**의 줄임말이에요. 마치 카톡에서 별표(`*굵게*`)를 쓰면 글자가 굵어지는 것처럼, 간단한 기호로 예쁜 문서를 만들 수 있는 형식이에요.

| Markdown 기호 | 결과 |
|---------------|------|
| `# 제목` | 큰 제목 |
| `**굵게**` | **굵게** |
| `- 항목` | 점 찍힌 목록 |

---

## 3️⃣ 상태 확인하기

```bash
git status
```

### 🤔 이게 뭐예요?

**"지금 내 작업 상태가 어때?"** 라고 Git에게 물어보는 거예요.

### 🔍 비유하자면?

**"카톡 안 읽은 메시지 확인하기"** 와 똑같아요!

```
📱 카톡 알림: "엄마 (1)" ← 안 읽은 메시지
🔧 git status: "TEST.MD (untracked)" ← 아직 처리 안 된 파일
```

### 🎨 색깔로 구분돼요

| 색깔 | 뜻 | 도시락 비유 |
|------|-----|-------------|
| 🔴 **빨강** | 새로 생긴 파일 / 수정된 파일 (아직 도시락통 밖) | 식탁 위에 음식 있음 |
| 🟢 **초록** | 도시락통에 담긴 파일 (보낼 준비 완료) | 도시락통에 담음 |
| ⚪ **흰색** | 아무 변화 없음 | 평소 상태 |

지금은 `TEST.MD`를 방금 만들었으니까 **빨간색**으로 표시될 거예요. Git이 "어? 새 파일이 생겼는데, 이거 보낼 거야?" 라고 묻는 거예요. 🤔

---

## 4️⃣ 스테이징 (도시락통에 담기)

```bash
git add TEST.MD
```

### 🤔 이게 뭐예요?

`add` = **"이 파일 보낼 거야! 도시락통에 담아줘"**

### 🍱 비유하자면?

**"보낼 물건만 골라서 박스에 넣기"** 예요.

```
🥪 김밥 ──┐
🍳 계란말이 ─┼──[ git add ]──→ 🍱 도시락통 (보낼 것만 모음)
🍕 안 보낼 피자 ──┘ (남겨둠)
```

### ❓ 왜 이 단계가 따로 있어요?

좋은 질문이에요! 🌟

수정한 파일 10개 중에 **5개만** 보내고 싶을 때가 있거든요. 예를 들면:

```
✏️ 오늘 수정한 파일들:
  - 숙제1.md ✅ (보낼 것)
  - 숙제2.md ✅ (보낼 것)  
  - 비밀일기.md ❌ (안 보낼 것!)
  - 게임공략.md ❌ (안 보낼 것!)
```

`git add 숙제1.md 숙제2.md` 라고 하면 **숙제만** 보내고 비밀은 안 보낼 수 있어요!

### 💡 모든 파일 한꺼번에 담고 싶다면?

```bash
git add .          # 점(.)은 "이 폴더 전부"라는 뜻
```

## 5️⃣ 커밋 (포장하고 라벨 붙이기)

```bash
git commit -m "TEST.MD 추가"
```

### 🤔 이게 뭐예요?

- `commit` = **"내 컴퓨터에 정식 저장하기"**
- `-m` = **m**essage (메시지)
- `"TEST.MD 추가"` = **이 작업이 뭐였는지 설명**

### 🎁 비유하자면?

**"포장하고 메모 붙이기"** 예요.

```
🍱 도시락통 ──[ git commit -m "..." ]──→ 🎁 메모 붙은 도시락
                                          📝 "친구 생일 도시락"
```

### ⚠️ 따옴표는 필수!

```bash
# ✅ 맞는 예시
git commit -m "TEST.MD 추가"

# ❌ 틀린 예시 (따옴표 없으면 에러!)
git commit -m TEST.MD 추가
```

따옴표는 **"여기서부터 여기까지가 메시지야!"** 라고 알려주는 표시예요. 카톡 보낼 때 메시지 입력창에 글자를 쓰는 것과 같아요. 📱

### 💌 좋은 커밋 메시지 쓰는 법

| 좋은 예시 ✅ | 나쁜 예시 ❌ |
|--------------|--------------|
| "TEST.MD 추가" | "ㅇㅇ" |
| "로그인 버그 수정" | "수정" |
| "메인 페이지 디자인 변경" | "ㅋㅋ" |

> 🎯 **3개월 뒤의 내가 봐도 알 수 있게!** 가 핵심이에요.

### 🏠 지금 파일은 어디에 있어요?

이 단계까지는 **내 컴퓨터 안**에만 저장돼 있어요. 아직 GitHub에는 안 갔어요!

```
✅ 내 컴퓨터 보관함: 저장 완료
❌ GitHub 창고: 아직 안 보냄
```

마치 도시락은 다 만들어서 **현관 앞에** 둔 상태예요. 아직 친구한테는 안 갔죠. 🚪

---

## 6️⃣ 푸시 (실제로 보내기!)

```bash
git push origin main
```

### 🤔 이게 뭐예요?

- `push` = **"보내!"**
- `origin` = **"GitHub의 내 저장소"** 라는 별명
- `main` = **메인 브랜치 (기본 작업 공간)**

### 🚚 비유하자면?

**"택배 기사님께 도시락 전달하기"** 예요!

```
🎁 현관 앞 도시락 ──[ git push ]──→ 🚚 택배 트럭 ──→ 🏢 GitHub
   (내 컴퓨터)                                        (인터넷 창고)
```

### 🌍 이제 무슨 일이 일어났나요?

1. ✅ **GitHub에 파일 업로드 완료**
2. ✅ **친구가 내 코드를 볼 수 있음**
3. ✅ **다른 컴퓨터에서도 받아갈 수 있음**
4. ✅ **혹시 컴퓨터가 망가져도 안전!**

브라우저에서 `https://github.com/내아이디/저장소이름` 에 가보면 **TEST.MD**가 올라가 있을 거예요! 🎉

## 🎬 처음부터 끝까지 따라하기

```bash
# 1️⃣ 작업할 폴더로 이동
cd /c/Users/nara/claude-code-guide

# 2️⃣ 새 파일 만들기
echo "# 테스트 파일" > TEST.MD

# 3️⃣ 상태 확인 (TEST.MD가 빨간색 untracked로 뜸)
git status

# 4️⃣ 스테이징 (도시락통에 담기)
git add TEST.MD

# 5️⃣ 커밋 (포장하기, 따옴표 필수!)
git commit -m "TEST.MD 추가"

# 6️⃣ 푸시 (GitHub로 보내기)
git push origin main
```

## ❓ 자주 묻는 질문

### Q1. 왜 이렇게 단계가 많아요? 그냥 한 번에 보내면 안 돼요?

**A.** 좋은 질문이에요! 이유가 있어요:

- 🛡️ **실수 방지**: 비밀 파일을 잘못 올리는 걸 막아줘요
- 🎯 **선택권**: 보낼 것만 골라서 보낼 수 있어요
- 📚 **기록**: 작업 단위마다 메모를 남길 수 있어요

마치 카톡 보내기 전에 한 번 더 읽어보는 것과 같은 안전장치예요. 💌

### Q2. `git status`에서 빨간색이면 뭐가 잘못된 거예요?

**A.** 아니에요! 빨간색은 **"아직 처리 안 된 새로운 변화"** 라는 뜻이에요. 잘못된 게 아니라, 그냥 "**도시락통에 안 담겼어요**" 라는 뜻이니까 `git add`만 해주면 초록색으로 바뀌어요. 🟢

### Q3. 커밋 메시지를 잘못 썼어요. 다시 쓸 수 있어요?

**A.** 네! 마지막 커밋이라면:

```bash
git commit --amend -m "올바른 메시지"
```

벌써 푸시까지 했다면? 🤔

```bash
git commit --amend -m "올바른 메시지"
git push -f origin main    # ⚠️ -f는 강제로 덮어쓰기
```

### Q4. `git push` 했는데 비밀번호를 물어봐요!

**A.** GitHub는 이제 비밀번호 대신 **Personal Access Token (PAT)** 이라는 특별한 비밀번호를 써요.

GitHub 사이트에서:
1. 우측 상단 프로필 → **Settings**
2. 좌측 메뉴 맨 아래 → **Developer settings**
3. **Personal access tokens** → **Tokens (classic)**
4. **Generate new token** 클릭해서 만들기

이걸 비밀번호 자리에 넣으면 돼요! 🔑

### Q5. 실수로 파일을 잘못 올렸어요!

**A.** 당황하지 마세요! Git은 **타임머신**이 있어요.

```bash
# 마지막 커밋 취소 (변경 내용은 유지)
git reset HEAD~1
```

자세한 건 **[Git 명령어 정리](./git-commands.md)** 를 확인하세요!
